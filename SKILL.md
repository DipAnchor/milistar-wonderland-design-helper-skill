# 千星奇域 · 玩法边界与实现查询方法论

## 何时触发

当用户的问题是以下之一时，阅读本 skill：

- 询问千星奇域的某个玩法/机制是否可实现（边界判断）
- 询问某类地图制作的技术方向和可行路径（实现指引）
- 需要分析千星奇域沙箱编辑器支持哪些系统（能力评估）
- 千星奇域地图制作工作流中，需要决策方案的可行性

## 版本一致性检查

每次使用本 skill 时，应确认 skill 记录的版本信息是否仍为当前版本：

1. **当前 skill 记录版本**：查阅 `references/milistar-updatelog.md` 中的最新版本日期，或查看 SKILL.md 中引用的版本号
2. **搜索确认最新版本**：通过搜索 `原神 最新版本号` 或 `千星奇域 最新版本` 确认当前游戏版本
3. **检查变更范围**：若最新版本号与 skill 记录的不一致，搜索该版本的更新公告，判断是否涉及千星奇域编辑器变更

**版本不匹配时**：提示用户还需要更新本 skill（"当前版本可能已更新，建议更新 skill 以获取最新边界信息"），但不阻塞当前回答——基于已知信息给出带版本标注的结论。
## 核心原则

本 skill 不存储具体的教程步骤或节点图配置细节。它的作用是：**提供一套检索方法论，让 AI 能自己找到答案**。

具体细节应通过以下方式获取：
1. 网页搜索：搜索关键词需要具体、结构化
2. 官方文档：米游社官方教程和文档页面**（动态页面，有本地缓存版本）**
3. 社区内容：B站视频、BWIKI、米游社论坛帖
4. 本地版本记录：`references/milistar-updatelog.md`（月之三~月之七更新汇总，手动维护）
5. **专用知识库 Skill（miliastra-knowledge）** — 通过 API 直接查询千星沙箱的节点说明、文档全文与语义检索。来源：skillhub 市场 `miliastra-toolbox` v1.0.3，后端 API 端点 `https://ugc.070077.xyz/`
6. 无头浏览器搜索：通过 agent-browser 打开米游社官方综合指南页面，在搜索框输入关键词检索（具体操作见 `references/sources.md`）
6. 用户导出MHTML：若当前环境无无头浏览器，可引导用户在浏览器中另存为「单个网页文件(.mhtml)」后发送给AI处理
7. 用户自身经验：用户玩过编辑器，可辅助验证和补充

```

│  ├─ 界面控件       → 自定义按钮/开关/图片/悬浮页/数字/进度条等（月之六起）

│  ├─ 变量与状态     → 数值/布尔/字符串变量，状态持久化

│  └─ 成就系统       → 自定义成就与条件
```

## 玩法可行性判定流程

当被问及千星奇域地图制作相关问题时（如"能不能做X""怎么做一个X""X节点图怎么搭"等），按以下流程判定和指引：

### 步骤 1：拆解玩法核心机制

把目标玩法拆成最基础的游戏机制单元。

**先判断呈现维度**：该玩法以3D为主（场景/角色/物理交互）、2D为主（界面/菜单/卡牌/HUD）、还是混合？

- **2D为主的玩法** → 优先以**界面控件（UI）方式拆解**，利用月之六起的界面控件（按钮/开关/图片/悬浮页/数字/进度条等）作为核心实现手段，减少对3D物件的依赖。
- **3D为主的玩法** → 以物件放置、节点图、触发器、战斗系统等3D场景能力为拆解方向。
- **混合型玩法** → 分拆2D部分（走UI）和3D部分（走场景），分别对应拆解。

例如：
- "RTS选兵" → 鼠标框选 + 单位选中 + 指令下达（3D为主）
- "塔防" → 路径寻路 + 敌人波次 + 防御塔攻击（3D为主）
- "回合制战斗" → 状态暂停 + 指令序列 + 结算（混合：3D战斗场景 + 2D指令菜单/血条HUD）
- "卡牌对战" → 卡牌展示 + 选择/拖拽 + 结算动画（2D为主：界面控件实现牌组+按钮+计数）

### 步骤 2：搜索平台现存案例

拆解完成后，**优先在社区平台搜索该玩法或类似玩法是否已有现成作品**。如果找到，可直接参考实现，无需再进行机制单元分析。

**搜索平台（按优先级）**：
1. **米游社官方教程搜索（无头浏览器）** — 在千星奇域综合指南页面搜索具体功能关键词（操作流程见 `references/sources.md`）
2. B站搜索API — `https://api.bilibili.com/x/web-interface/search/all/v2?keyword=千星 [玩法名]`（结构化结果，可直接抓取）
3. B站网页搜索 `千星奇域 [玩法名]`
4. 米游社千星奇域专区 `https://www.miyoushe.com/ys/topicDetail/2168`
5. 通用网页搜索 `千星 [玩法名]`

**搜索策略**：
- 同时用机制名和类比搜索（如搜"MOBA"也搜"降临者峡谷"，搜"射击"也搜"枪械精通"）
- 如果找到视频/教程，记录 BV 号或链接作为证据

**判断**：
- ✅ **有现成案例** → 直接进入实现阶段，参考现有作品
- ⚠️ **有部分相似案例** → 进入下一步对照能力表，确认差异部分
- ❌ **无现成案例** → 进入下一步对照能力表，从零分析

### 步骤 3：对照系统能力拓扑

检查每个机制单元是否落在千星沙箱已知能力范围内。如果步骤2找到了现成案例，此步仅检查案例未覆盖的差异部分。每项能力标注了**证据来源**——已确认的来自公开资料，未确认的需要实时检索。

| 能力类别 | 状态 | 证据来源 | 备注 |
|---------|------|---------|------|

### 步骤 4：搜索确认

对于不确定性高的能力，使用以下检索策略。**关键原则：所有信息必须验证时效性，不采信过时资料。**

**搜索注意事项**：

1. **版本时效** — 注意资料的发布时间和标注的适用版本。千星奇域随版本更新迭代，旧教程可能因界面/功能变更已不适用。优先采信当前版本（v6.6 月之七）范围内的资料。
2. **缩写搜索** — 笼统搜索时优先用 `千星` 替代 `千星奇域` 或 `千星沙箱`，覆盖面更大。定位到具体内容后再用全称缩小范围。
3. **类比搜索** — 不只搜机制名，同时用该玩法的知名游戏作为关键词搜索。例如：
   - 搜"打撤/搜刮撤离" → 同时搜 `塔科夫` `鸭科夫`
   - 搜"狼人杀社交推理" → 同时搜 `太空狼人杀` `Among Us`
   - 搜"自走棋" → 同时搜 `云顶之弈` `金铲铲之战`
   - 搜"糖豆人闯关" → 同时搜 `糖豆人` `Fall Guys`
   这有助于找到更多实现参考和节点图思路。
4. **多平台覆盖** — 交替搜索B站、通用搜索引擎、米游社、BWIKI，不同平台的内容侧重点不同。

**搜索关键词模板**：
- `千星 [功能名] 实现`
- `千星 [机制名] 节点图`
- `千星 [玩法类型] 教程`
- `原神 UGC [功能] 怎么做`
- `千星 [游戏名] 复刻`（类比搜索）

**优先搜索源**：
0. **本地更新日志** `references/milistar-updatelog.md` — **优先查阅**，月之三~月之七版本变更记录，许多能力是否已添加可直接在此找到答案
1. **专用知识库 Skill（miliastra-knowledge）** — 调用其 `get_node_info`、`list_documents`、`get_document`、`rag_search` 四个工具，直接查询千星沙箱知识库 API。来源：skillhub 市场 `miliastra-toolbox` v1.0.3，API 端点 `https://ugc.070077.xyz/`
2. **米游社官方教程搜索（无头浏览器）** — 使用 agent-browser 搜索米游社综合指南页面（操作流程见 `references/sources.md`）
3. B站搜索 `千星 [关键词]` — 视频教程，实操演示
3. B站搜索API `https://api.bilibili.com/x/web-interface/search/all/v2?keyword=[关键词]` — 绕过网页渲染限制，直接获取结构化搜索结果
4. 米游社千星奇域专区 — 社区讨论与作品分享
5. 千星奇域BWIKI — 系统数据百科
6. 通用网页搜索 — 兜底补充

### 步骤 5：判断结论

- **全部机制单元都有✅确认证据** → 可行，可直接进入实现阶段
- **任意机制单元证据不足或标记为⚠️** → **必须实时搜索确认**，有社区实现案例则可行，否则标记为存疑
- **有关键机制单元标记为❌** → 不可行，或需寻找替代实现方案（如用节点图模拟替代原生系统，或改变玩法设计绕开限制）

> 结论为建议性，最终可行性也受用户编辑经验、硬件配置和节点图熟练度影响。

## 地图制作方向指引

当被问及"怎么做一个xx类型的地图"时，给出以下方向的指引：

1. **类型匹配**：明确地图属于哪一类（冒险闯关/对抗竞技/模拟经营/多人派对/解谜）
2. **核心机制**：列出该类型需要哪几个核心机制单元
3. **实现路径**：搜索同类教程，参考节点图结构
4. **边界提醒**：指出可能的限制和替代方案
5. **测试建议**：利用编辑器内置测试功能迭代

## 与外部工作流协作

本 skill 设计为可被其他AI工作流调用的"查询模块"。

当主工作流（如"千星奇域地图设计助手"）需要判断某个玩法是否可实现时：
1. 调用本 skill 的可行性判定流程 → 得到"可行/存疑/不可行"结论
